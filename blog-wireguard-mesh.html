How I Built a WireGuard Mesh Across 7 Nodes for $6/month

Hub-and-spoke WireGuard connecting 5 Raspberry Pis behind NAT to 2 cloud servers. 12 tunnels, full end-to-end encryption, zero monthly cost for the VPN itself.

The problem

Five Raspberry Pis sit behind a residential NAT with a dynamic IP. Two DigitalOcean droplets have static public IPs in NYC. Every node needs to talk to every other node securely. SSH, database replication, AI inference requests, NATS pub/sub messages, DNS zone transfers — all of it encrypted, all of it routable.

The standard answer is Tailscale or Zerotier. Both are good. Both cost money per seat. Both add a dependency on someone else's coordination server. I wanted something I own completely.

The topology

WireGuard runs as a kernel module on every node. The topology is hub-and-spoke with Gematria (the NYC3 droplet) as the hub. Every Pi connects to Gematria through a persistent WireGuard tunnel that punches through NAT. The Pis also have peer-to-peer tunnels to each other for LAN traffic.

The result: 12 bidirectional tunnels forming a full mesh. Any node can reach any other node by IP address, whether they're on the same LAN, in a data center, or on opposite sides of a NAT.

Alice (Pi 4, gateway) — 10.0.0.1
Cecilia (Pi 5, AI) — 10.0.0.2
Octavia (Pi 5, DevOps) — 10.0.0.3
Lucidia (Pi 4, web) — 10.0.0.4
Aria (Pi 4, backup) — 10.0.0.5
Gematria (DO NYC3, edge) — 10.0.0.10
Anastasia (DO NYC1, DNS) — 10.0.0.11

The setup

On each node: apt install wireguard. Generate keys: wg genkey | tee private.key | wg pubkey > public.key. Write /etc/wireguard/wg0.conf. Enable: systemctl enable --now wg-quick@wg0.

The entire config per node is about 30 lines. It lists the node's own address, its private key, and a [Peer] section for each other node with their public key, allowed IPs, and endpoint (for public nodes) or no endpoint (for NAT'd nodes that connect outbound).

The Pis use PersistentKeepalive = 25 to maintain the NAT mapping. Without it, the tunnel drops after the NAT table entry expires (usually 30-120 seconds of inactivity).

What flows through it

SSH: I can SSH from my Mac to any Pi by its WireGuard IP. ssh blackroad@10.0.0.3 reaches Octavia regardless of what the residential IP is doing.

Gitea replication: Gitea on Octavia (10.0.0.3:3100) is accessible from all nodes. Push from any machine.

Ollama inference: Chat requests arrive at Gematria (public), route through WireGuard to Cecilia (10.0.0.2:11434), get inference results, and return. The user never knows they're talking to a Raspberry Pi.

DNS: PowerDNS zone transfers between Lucidia (primary) and Gematria (secondary) happen over WireGuard. No unencrypted DNS data on the wire.

NATS: The NATS cluster on Octavia uses WireGuard IPs for all inter-node communication. Pub/sub messages between agents are encrypted in transit.

What it cost

WireGuard itself is free and open source. It's in the Linux kernel. There's no license, no per-seat fee, no coordination server to pay for.

The only cost is the two DigitalOcean droplets that serve as public endpoints: Gematria at $24/month and Anastasia at $6/month. But those droplets exist for Caddy TLS and DNS — the WireGuard tunnels are a side benefit. The VPN mesh itself costs $0.

What I learned

WireGuard is absurdly simple. The total config across all 7 nodes is under 250 lines. There's no certificate authority, no key exchange protocol, no daemon to manage. It's stateless, fast, and it just works.

NAT traversal is solved. PersistentKeepalive handles everything. I haven't had a single tunnel drop in months.

Full mesh is worth it. Hub-and-spoke would be simpler but every cross-Pi request would bounce through Gematria in NYC. Full mesh means Cecilia→Octavia goes direct on the LAN at gigabit speed instead of round-tripping to a data center.

The kernel module matters. WireGuard runs in kernel space, not userspace. On a Pi 4, throughput is about 400 Mbps. That's more than enough for anything the fleet needs to do.

12 tunnels. 7 nodes. 0 monthly cost for the VPN itself. Every packet encrypted end-to-end. If you're self-hosting anything, WireGuard should be the first thing you set up.
